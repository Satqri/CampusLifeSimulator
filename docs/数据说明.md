# 大学生活模拟事件系统 — 变量、数值与结局完整说明

---

## 一、玩家属性体系总览

游戏中共有 **5 个基础属性** + **1 个核心隐藏属性** + **20+ 个常规隐藏变量** + **多个布尔/枚举型状态标记**。

---

### 1.1 五大基础属性

| 变量名 | 中文含义 | 范围 | 说明 |
|--------|----------|------|------|
| `academic` | 学术 / 知识 | 0~100 | 代表学习成绩、知识储备和解题能力。学习、复习、做项目等增加；逃课、划水等间接影响。 |
| `social` | 社交能力 | 0~100 | 代表人际关系、沟通技巧和声望。帮助他人、参与社团、得体社交增加。 |
| `energy` | 能量 / 体力 | 0~100 | **核心消耗资源**。任何日常行动都会消耗，吃饭/睡觉/零食恢复。降至 0 触发强制结局。 |
| `stress` / `san` | 压力 / 心态 | 0~100 | **反向指标**——越高代表压力越大。JSON 中用 `san`；设计文档中用 `stress`。面对尴尬、出糗、点名、失败时增加（恶化）；放松、成功时减少（改善）。≥100 触发强制结局。 |
| `gold` | 金钱 | 0~999 | 手头现金。打工、加班挣钱；买东西、请客、赔差价花钱。 |

### 1.2 核心隐藏变量：健康指数 (healthIndex)

| 变量名 | 初始值 | 说明 |
|--------|--------|------|
| `healthIndex` | 100 | 衡量长期生活习惯的底线指标，不可直接查看。 |

| 阶段界限 | 游戏内提示 / 效果 |
|----------|-------------------|
| `> 60` | 健康：无提示，身体状况良好 |
| `≤ 60` | 亚健康警告："你最近总是感觉头晕乏力，也许该少熬点夜，或者去锻炼一下了。"（隐藏效果：每日基础 Energy 恢复量减少） |
| `≤ 30` | 疾病预警："你的心脏时不时传来刺痛，呼吸也变得沉重。"（隐藏效果：屏幕边缘红色暗角） |
| `≤ 0` | **强制结局**：立即触发【因病休学（医院结局）】 |

### 1.3 常规隐藏变量（数值型 Int）

在 14 天的流程中，除特别说明外，累积极值通常在 20~40 左右。

#### 1.3.1 课程与点名相关

| 变量名 | 含义 | 备注 |
|--------|------|------|
| `lateCount` | 迟到次数 | 影响老师点名与相关副结局 |
| `skipClassCount` | 缺课次数 | 影响老师信任度与主结局判定 |
| `teacherTrust` | 老师信任度 | 上课增加，逃课/缺勤减少 |
| `classAttendCount` | 到课次数 | 记录玩家按时/补救进入课堂的次数 |
| `rollCallCount` | 点名触发次数 | 用于课程点名链副结局判定 |
| `rollCallSavedCount` | 点名救场成功次数 | 包含同学圆场与赶回教室成功 |
| `absencePenaltyCount` | 缺勤惩罚次数 | 被正式记缺勤或补救失败时增加 |
| `returnClassCount` | 赶回教室次数 | 影响"迟到补锅人/早八冲刺王"称号 |

#### 1.3.2 同学互助相关

| 变量名 | 含义 | 备注 |
|--------|------|------|
| `friendBond` | 同学羁绊值 | 决定同学互助事件的触发与成功率；正面互动增加，拒绝/冷漠减少 |
| `friendHelpCount` | 同学互助次数 | 帮助同学或被同学帮助时增加 |
| `friendRefuseCount` | 拒绝互助次数 | 影响"关系冷淡/已读不回的人"称号 |
| `friendRollCallHelpCount` | 同学点名救场次数 | 记录同学在点名链中成功帮忙 |
| `friendReviewCount` | 考前互助复习次数 | 影响考前伙伴与资料共享者称号 |

#### 1.3.3 社团相关

| 变量名 | 含义 | 备注 |
|--------|------|------|
| `clubActivityCount` | 社团参与次数 | 影响社团路线结算 |
| `clubContribution` | 社团贡献度 | 影响社团主心骨、展示担当等称号 |
| `clubRelation` | 社团关系值 | 影响社团分歧和展示结果 |
| `clubShowcaseScore` | 社团展示分 | 社团展示节点的最终表现 |

#### 1.3.4 大创比赛相关

| 变量名 | 含义 | 备注 |
|--------|------|------|
| `innovationProgress` | 大创进度 | **极值上限 100**；推进方案、熬夜、会议等增加 |
| `innovationTeamTrust` | 大创团队信任 | 信任低更容易触发队友失联等事件 |
| `innovationDefenseScore` | 大创答辩分 | 用于大创现场答辩与结果结算 |
| `innovationCrisisCount` | 大创危机次数 | 中期危机、Demo 翻车、队友失联时增加 |

#### 1.3.5 日常积累相关

| 变量名 | 含义 | 备注 |
|--------|------|------|
| `lateNightLevel` | 熬夜程度 | 熬夜、打游戏、加班等增加；睡觉减少 |
| `partTimeCount` | 兼职次数 | 决定是否触发"打工皇帝"结局 |

| `storeTrust` | 便利店排班信任 | 影响后续夜班兼职与便利店事件收益；≤ -5 则夜班不可触发 |

### 1.6 逻辑换算说明

设计文档中的表述与 JSON 中的判定对应关系：

| 设计文档表述 | JSON 实际判定 |
|-------------|---------------|
| SAN 较高 / 临场稳定 | `stress ≤ 50` |
| SAN 较低 / 心态崩溃 | `stress ≥ 70` |
| Expression | `expressionUnlocked = true` 或 `social ≥ 60` |
| Logic | `logicUnlocked = true` 或 `academic ≥ 60` |
| Research | `researchUnlocked = true` 或 `academic ≥ 60` |
| Campus Intel | `campusIntelUnlocked = true` |

### 1.7 流程判定与数值边界

| 规则 | 处理方式 |
|------|----------|
| 基础属性越界 | `energy`、`stress`、`academic`、`social` 结算后统一限制在 0~100 |
| 金钱越界 | `gold` 结算后统一限制在 0~999；若支付选项会导致 `gold < 0`，则该选项不可选 |
| 隐藏计数越界 | 次数型隐藏变量最低为 0；`innovationProgress` 限制在 0~100 |
| 行动结算顺序 | 先检查选项门槛 → 结算属性变化 → 限制数值范围 → 检查强制结局与任务链结果 |
| Energy 不足 | 有 Energy 门槛的选项不可选；无门槛行动若结算后 `energy ≤ 0`，立即触发【崩溃退学】 |
| Stress 爆表 | 任意行动结算后若 `stress ≥ 100`，立即触发【崩溃退学】 |
| 健康归零 | 任意行动结算后若 `healthIndex ≤ 0`，优先触发【因病休学】 |
| 点名事件重复计数 | 课程点名链与同学互助链共用同一次点名；同一事件只允许 `rollCallCount +1` |
| 大创未报名 | `innovationJoined = false` 时，大创常规交互不增加 `innovationProgress` |
| 欠人情未偿还 | 第 14 天若 `owedFavor = true` 且 `friendBond < 4`，额外 `friendRefuseCount +1` |
| 便利店信誉过低 | 若 `storeTrust ≤ -5`，后续便利店夜班兼职不可触发，直到通过正向事件修复到 -2 以上 |

## 二、事件步骤类型（Step Types）

每个事件由多个步骤组成，步骤之间通过 `then`（成功时跳转）和 `else`（失败时跳转）串联。

| 类型 | 说明 | 关键字段 |
|------|------|----------|
| `display` | 纯展示文本，向玩家描述当前场景 | `title`（标题）, `body`（正文）, `then`（下一步） |
| `choice` | 玩家选择分支，提供 2-5 个选项 | `options`（选项数组，每项含 `text` 和 `then`） |
| `check` | 条件判断节点，根据玩家属性决定成功/失败 | `conditions`（条件数组）, `require`（`all` 全部满足 / `any` 任一满足）, `then`（成功跳转）, `else`（失败跳转） |
| `outcome` | 结局节点，事件结束，属性发生变化 | `delta`（基础属性增减对象）, 可选的 `time_advance`（时间推进） |
| `random_check` | 随机判定节点，按概率决定成功/失败 | `probability`（成功概率 0-100）, `success`（成功跳转）, `failure`（失败跳转） |

## 三、条件判断（Check Conditions）

### 3.1 属性条件（stat check）

```json
{ "kind": "stat", "stat": "academic", "op": ">=", "value": 50 }
```

- `kind`: `"stat"` 表示属性判定
- `stat`: 属性名（`academic`, `social`, `energy`, `san`, `gold`）
- `op`: 比较运算符（`>=` 大于等于, `>` 大于, `<` 小于, `true` 表示仅检查属性存在且大于 0）
- `value`: 阈值

### 3.2 位置条件（location check）

```json
{ "kind": "location", "place": "classroom" }
```

检查玩家当前是否在指定位置。

### 3.3 组合条件（require）

- `"require": "all"` — 所有条件必须同时满足才算通过
- `"require": "any"` — 任一条件满足即可通过

## 四、触发方式（Trigger Types）

| 触发类型 | 说明 | 示例 |
|----------|------|------|
| `interaction` | 玩家与特定场景交互时触发，通过 `action_id` 指定交互点 | `"action_id": "library_desk"` 在图书馆书桌触发 |
| `time_schedule` | 由时间系统自动触发，通过 `method` 指定触发方式 | `"method": "crossed_class_time"` 上课时间到了但玩家不在教室时触发 |

## 十、特殊机制总结

### 10.1 连锁事件

`friend_help_chain.json`、`club_chain.json`、`innovation_chain.json` 包含多个连续事件。前一个事件的结果会影响：
- 后续事件中的属性值和可选路径
- 隐藏变量的累积（如 friendBond、clubContribution、innovationProgress）
- 是否触发某些后续事件（如帮过同学后食堂回礼才会出现）

### 10.2 随机点名

`class_attendance` 使用 `random_check` 类型，以 45% 概率判定老师是否点名。若不在教室且被点名，触发一系列补救选项。点名事件与同学互助链联动（friendBond 越高，圆场成功率越高）。

### 10.3 时间推进

部分 `outcome` 节点包含 `time_advance` 字段（如 `"time_advance": 45`），表示该选择会推进游戏内时间（单位：分钟）。

### 10.4 空 Delta

有些结局的 `delta` 为空对象 `{}`。这表示该选择不会改变基础属性值，但隐藏变量可能仍然发生变化（如 teacherTrust、friendBond 等），它仍然是一个有意义的叙事选择。

### 10.5 强制结局中断

- `healthIndex ≤ 0`：优先触发【因病休学】
- `stress ≥ 100` 或 `energy ≤ 0`：触发【崩溃退学】
- 以上两个条件在任何行动结算后检查，一旦触发立即中断游戏

## 十一、设计哲学

1. **没有绝对的最优解**：每个选择都有代价。高收益往往伴随着高风险或高消耗。
2. **属性互斥**：提高学术可能需要消耗体力；维持社交可能需要承受压力；赚钱可能需要牺牲社交评价。
3. **san/stress 是压力值**：大多数"失败"结局会增加压力，而成功或人际温暖会减少压力。≥100 直接退学。
4. **健康是隐藏底线**：healthIndex 无形中惩罚熬夜、吃垃圾食品等不良习惯，≤0 直接医院结局。
5. **连锁反应**：帮过的人可能会在关键时刻回报你；在社团里的投入会影响最终展示的表现；大创的每一步选择都会累积到最终答辩。
6. **真实感**：不是所有选择都能成功——属性不够时强行尝试可能导致更糟糕的结果。隐藏变量的累积效应让短期选择产生长期后果。
