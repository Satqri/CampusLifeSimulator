# 校园生活模拟器 - 属性、隐藏变量与结局设计

本文件保留设计入口；当前权威实现规范见 [[数据说明]]。

## 核心约定

- `san` 表示心态稳定度，越高越好。
- `stress` 只作为派生概念存在，换算为 `100 - san`。
- `healthIndex` 是长期健康底线；可见 `health` 是它的 0~100 显示投影。
- 事件 JSON 可以同时写 `delta` 和 `hidden_delta`。
- `friendStage`、`clubStage`、`innovationStage` 是阶段状态，按赋值处理，不按累加处理。

## 即时结局

| 结局 | 条件 |
| --- | --- |
| 因病休学 | `healthIndex <= 0` |
| 崩溃退学 | `lowEnergyDays >= 3` 或 `san <= 0` |

## 最终结算

最终主结局和称号由 `assets/config/events/endings.json`、`assets/config/events/titles.json` 与 `SettlementResolver` 共同决定。长期路线包括课程点名、同学互助、社团、大创、兼职、锻炼、图书馆、食堂和宿舍娱乐等。

## 数值维护

新增或修改事件时，先查阅 [[数据说明]] 中的“数值标定原则”和“统一结算顺序”。不要再使用旧表述“`san` 越高代表压力越大”；压力上升应写为 `san` 负数。
